# KinoFiksMontaj Bot

Telegram-бот, который принимает видео, нарезает из него короткий трейлер
по сменам сцен и накладывает автоматически сгенерированные субтитры.

## Как это работает

1. Пользователь шлёт видео боту
2. `PySceneDetect` находит моменты смены сцен в видео
3. Из этих моментов собирается короткий клип нужной длины (по умолчанию 60 сек)
4. `faster-whisper` распознаёт речь в клипе и генерирует `.srt` субтитры
5. `ffmpeg` "вшивает" субтитры в кадр
6. Готовое видео отправляется пользователю обратно

## Локальный запуск (для теста на своём компьютере)

1. Установи Python 3.11+ и ffmpeg (`sudo apt install ffmpeg` на Linux,
   `brew install ffmpeg` на Mac, на Windows скачай с ffmpeg.org)
2. Склонируй репозиторий и зайди в папку:
   ```bash
   git clone <ссылка_на_твой_репозиторий>
   cd kinofiks_bot
   ```
3. Создай виртуальное окружение и установи зависимости:
   ```bash
   python -m venv venv
   source venv/bin/activate   # на Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
4. Скопируй `.env.example` в `.env` и впиши туда токен бота:
   ```bash
   cp .env.example .env
   ```
5. Запусти бота:
   ```bash
   python bot.py
   ```

## Деплой на Render

Render не подхватывает `.env` файл сам - переменные окружения нужно
вписать вручную в панели Render (см. ниже).

1. Залей проект на GitHub (см. раздел ниже "Как залить на GitHub")
2. Зайди на https://render.com и зарегистрируйся / залогинься
3. New + → "Background Worker" (НЕ Web Service! Боту не нужен открытый порт,
   он сам подключается к Telegram)
4. Подключи свой GitHub репозиторий
5. Render должен сам обнаружить `Dockerfile` и предложить Docker-окружение -
   подтверди это
6. В разделе "Environment" добавь переменные:
   - `BOT_TOKEN` = твой токен от @BotFather
   - `ADMIN_IDS` = твои Telegram ID через запятую
7. Нажми "Create Background Worker" - Render соберёт Docker-образ и запустит бота
8. В логах должно появиться "Бот запускается…" - значит всё ок

### Важно про бесплатный тариф Render

Бесплатный план Render даёт мало RAM и CPU. Обработка видео (особенно
распознавание речи через Whisper) - тяжёлая задача. Если видео длинное
или его много одновременно, бесплатный воркер может упасть по памяти
или работать очень медленно. Для серьёзного использования стоит
посмотреть платные планы Render с большим объёмом RAM (Standard/Pro),
либо перенести обработку видео на отдельный VPS.

## Как залить проект на GitHub

Если ты никогда не работал с Git:

1. Зарегистрируйся на https://github.com
2. Создай новый репозиторий (кнопка "New repository"), назови его
   например `kinofiks-bot`, оставь его приватным (Private) - в нём
   будет код, но без секретов (токен туда не попадёт благодаря `.gitignore`)
3. На своём компьютере, в папке проекта, выполни:
   ```bash
   git init
   git add .
   git commit -m "Первая версия бота"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_НИК/kinofiks-bot.git
   git push -u origin main
   ```
4. Готово - теперь Render сможет подключиться к этому репозиторию

## Известные ограничения

- Telegram Bot API ограничивает размер файлов, которые бот может
  скачать/отправить, обычно 50 МБ через стандартный API. Для больших
  файлов нужно поднимать свой Local Bot API Server (см. документацию
  Telegram), это отдельная настройка, не включённая в этот проект.
- Whisper модель `small` выбрана как баланс между скоростью и точностью
  на CPU. Можно поменять на `tiny` (быстрее, менее точно) или `medium`/`large`
  (точнее, но в разы медленнее и прожорливее по памяти) в `video_processor.py`.
- Алгоритм выбора "интересных" моментов сейчас базовый - находит смены
  сцен и берёт равномерную выборку. Это не "умный" AI-монтаж, а скорее
  заготовка, которую можно улучшать (например, добавить анализ громкости
  аудио, лиц в кадре, движения и т.д.)
